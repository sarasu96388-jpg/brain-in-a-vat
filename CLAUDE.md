# CLAUDE.md — 项目主控文件

> 每个会话启动时必读。本文件定义了项目的背景、目标、规则和当前状态。

## 项目概述

本项目是一个 **"AI 多人格协作架构"** 的实验性仓库，灵感来源于 B.I.A.V. Studio 的「缸中之脑计划」（Brain in a Vat Project）。

核心目标：验证一个 **零代码基础的人** 如何仅通过与 AI 对话，驱动完成完整的项目工作流——包括数据抓取、信息分析、文档交付和自动化流水线。

## 架构设计

系统分为三层：

| 层级 | 工具 | 职责 |
|------|------|------|
| **前台** (Front Desk) | claude.ai 聊天 | 与用户对话、分析、交付 PDF/docx |
| **中台** (Middle Office) | Claude Code 终端 | 执行代码、自动化脚本、协调子项目 |
| **后台** (Back Office) | GitHub 仓库 | 持久化存储：结构化记忆、可调用资产 |

**关键约束**：前台和中台不能直接通信，所有信息必须通过后台（本仓库）流转。

## 工作规则

1. **每次会话开始时**：读取本文件和 `memory/decisions.md` 了解上下文
2. **做决策时**：将确认的结论写入 `memory/decisions.md`
3. **交付成果时**：将文件放入 `deliverables/` 目录
4. **产生新数据时**：存入 `assets/data/` 并更新索引
5. **保持简洁**：用自然语言沟通，避免过度工程化
